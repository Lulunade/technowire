De ontwikkeling van programmeertalen (van 0 tot NU)

Machinecode:
Machinetaal is een taal waarin instructies geschreven zijn die de processor van een computer direct kan uitvoeren. Een programma bestaat in machinetaal uit een reeks woorden, meestal van 8 (een byte), 16, 32 of 64 bits. Elk woord staat voor een specifieke actie die de processor dient uit te voeren. Wanneer de computer in het programma dat hij aan het uitvoeren is bijvoorbeeld de byte 00001010 tegenkomt dan kan dit betekenen dat hij de inhoud van register 1 bij register 0 moet optellen.

Assembleertalen:
Assembleertaal of assembly is een low-level programmeertaal die nauwelijks meer dan een symbolische weergave van machinetaal is. Elke regel in de assembleertaal komt overeen met één enkele instructie, zij het dat de vertalende assembler nog wel ondersteuning biedt in de vorm van symbolische namen (labels) voor geheugenlocaties die gebruikt worden voor constanten of variabelen, en voor sprongopdrachten, in plaats van fysieke adressen. Voor de instructiecode gebruikt men mnemonics: zo kan de instructie "load accumulator" weergegeven worden door de mnemonic LDA, "store accumulator" door STA en "optellen" door ADD. De eenvoudigste mnemonic is NOP, namelijk No OPeration - er wordt niets gedaan.

Hogere programmeertalen:
Een programmeertaal waarvan de instructies in bijna-menselijke taal zijn geschreven, waardoor een programma gemakkelijker te leren en te begrijpen is. Enkele hogere programmeertalen zijn: FORTRAN, COBOL, BASIC, Pascal en Ada. Ze worden ook wel programmeertalen van de derde generatie genoemd.

Vierde generatie talen:
Vierde generatie programmeertaal is een programmeertaal die specifiek is ontworpen met een bepaald doel in het achterhoofd. Meestal is de 4GL taal gebaseerd op een dichter bij de computer staande programmeertaal en zodanig gestructureerd dat kennis van deze "lagere" programmeertaal niet nodig is, om toch toepassingen te kunnen schrijven.

Object georiënteerde talen:
Object georiënteerd programmeren is een manier van programmeren waarbij logica en gegevens worden georganiseerd rondom objecten. Aan de hand van een object wordt een datastructuur gecreëerd met verschillende gegevens en methodes. Dit programmeermodel geldt al vele jaren als de standaard binnen softwareontwikkeling. De manier waarop logica wordt opgedeeld rondom objecten zorgt ervoor dat code leesbaar en beheerbaar blijft (in tegenstelling tot sequentieel opgebouwde code).

Naam: Fortran
Jaartal: 1957
Ontwikkelaar: John Backus

FORTRAN is een programmeertaal die vooral gebruikt wordt in wetenschappelijke gemeenschappen. De naam komt van FORmula TRANslation en de bedoeling ervan was om ingewikkelde wiskundige sommen met meer gemak dan ASSEMBLY op te kunnen lossen.

Naam: Cobol
Jaartal: 1959
Ontwikkelaar: Dr. Grace Murray Hopper

C.O.B.O.L staat voor 'Common Organization Business-Oriented Language' en wordt veel gebruikt in de financiële service industrie. Het wordt al veel gebruikt sinds de jaren 60'. C.O.B.O.L is ontworpen voor gebruik business applicaties en het was gemaakt met de invoering van 'fixed-point arithmetic' Een van de voordelen van C.O.B.O.L is het rekenen met zakelijke variabelen en het geven van een consequent accuraat antwoord.

Naam: Basic
Jaartal: 1964
Ontwikkelaar: John G. Kemeny en Thomas E. Kurtz

De afkorting B.A.S.I.C stands for Beginner's All-Purpose Symbolic Instruction Code. Het nuttige van B.A.S.I.C is dat het een redelijk simpele syntax heeft en daardoor een goede manier om bijvoorbeeld te leren programmeren, alhoewel de taal niet zo veel meer wordt gebruikt als voorheen in de ontwikkeling van huidige applicaties.

Naam: C
Jaartal: 1969
Ontwikkelaar: Dennis Richie

C is de directe opvolger van de B programmeertaal (wat zelf een kleinere versie was van BCPL). Het UNIX besturingssysteem is uiteindelijk herschreven in C om het besturingssysteem compacter en flexibeler te maken. Het feit dat UNIX zo compact was, maakte dat C en UNIX zo populair waren in de tijd dat ze uitkwamen. Dit zorgte er namelijk voor dat ze een besturingssysteem konden maken die draaide op C en dat de applicatie voor dat besturingssysteem in dezelfde taal konden worden geschreven.

Naam: Pascal
Jaartal: 1970
Ontwikkelaar: Niklaus Wirth

Pascal is een influentiele programmeerlanguage vernoemd naar de wiskundige Blaise Pascal. Het is ontworpen door Niklaus Wirth in 1968 voor een onderzoeksproject in de categorie 'compiler theory'.

Naam: C++
Jaartal: 1983
Ontwikkelaar: Bjarne Stroustrup

C++ is een extensie van de C programmeertaal. C++ werd ontwikkeld met ondersteuning voor 'system programming' en 'embedded, resource-constrained software and large systems'. De taal heeft als kernpunten: prestatie, efficiëntie en flexibiliteit. Hiernaast biedt het ook de optie voor dingen als: Object Georiënteerd Programmeren en Funcioneel Programmeren.

Naam: Perl
Jaartal: 1987
Ontwikkelaar: Larry Wall

Wanneer we praten over Perl hebben we het over Perl 5 en niet over Perl 6, want dat heet sinds 16 oktober 2019 Raku. Perl heeft geen officiele afkorting. Perl valt onder Vrije Software en werd ontwikkeld op het UNIX besturingssysteem. Op de meeste besturingssystemen is Perl te gebruiken, maar het wordt het meest op UNIX en systemen gebasseerd op UNIX (zoals linux) gebruikt. Wikipedia werkte tot aan januari 2002 op een CGI script die was geschreven in Perl.

Naam: Python
Jaartal: 1991
Ontwikkelaar: Guido van Rossum

Python is een high-level programmeertaal, wat inhoudt dat de taal erg makkelijk te lezen en te begrijpen is en daardoor veel verder van de computer's code afstaat dan een low-level taal zoals C. Nog een specialiteit van Python is het gebruik van witruimte om aan te tonen dat code bij elkaar hoort. Python heeft twee verschillende versies: Python 2 en Python 3, de code van deze versies zijn niet een op een verwisselbaar en daardoor zou een Python 2 programma voor Python 3 moeten worden herschreven als dit nodig is.

Naam: Ruby
Jaartal: 1993
Ontwikkelaar: Yukihiro "matz" Masumoto

Ruby is een object-georiënteerde programmeertaal. De taal leent een groot deel van de syntax en principes van Perl, het is een soort object-georiënteerde versie van Perl. Ruby leent ook een deel van Lisp en Smalltalk. Alhoewel ruby niet veel heeft geleend van de programmeertaal Python, heeft het lezen van de Python code de ontwikkelaar geholpen in het schrijven van Ruby.

Naam: PHP
Jaartal: 1995
Ontwikkelaar: Rasmus Lerdorf

P.H.P is een afkorting en staat voor PHP: Hypertext Preprocessor. In haar essentie zorgt P.H.P ervoor dat een statische website dynamisch wordt. De reden waarom het een Preprocessor is, komt doordat het code inlaadt voordat de HTML pagina wordt geladen. Hiermee kun je blogs maken, op afstand bestuurbare hardware bedienen of een krachtige website zoals wikipedia of wikibooks laten draaien. Hiervoor heb je wel een database applicatie nodig, zoals MySQL.

Naam: Java
Jaartal: 1996
Ontwikkelaar: James Gosling

Java wordt over het algemeen gezien als klassengerichte en object georiënteerde taal, alhoewel het niet volstrekt object georiënteerd is omdat het primitieve types gebruikt. De taal is onder andere ontworpen met de bedoeling om niet te veel van kleinere programma's afhankelijk te zijn. Het doel van Java is om gecompileerde Java code overal te kunnen uitvoeren. Dat houdt dus in dat een Java programma gemaakt in Windows ook moet werken op Linux of MacOS. De syntaxis van Java lijkt veel op dat van C en C++, maar heeft minder low-level aspecten dan ze allebei.

Naam: JavaScript
Jaartal: 1997
Ontwikkelaar: Brendan Eich

JavaScript is een veelgebruikte scripttaal om webpagina's interactief te maken en webapplicaties te ontwikkelen. Naast HTML en CSS is JavaScript een van de kerntechnologieën van het wereldwijde web. Netscape heeft in een poging om JavaScript als een officiële norm erkend te krijgen en er toch zelf voldoende zeggingskracht over te behouden de taal laten normeren door de European Computer Manufacturers Association (ECMA), waarbij de naam om redenen betreffende het merkrecht is gewijzigd in ECMAScript.

Naam: Ruby On Rails
Jaartal: 2005
Ontwikkelaar: David Heinemeier

Ruby on Rails is een opensource-webapplicatieframework, geschreven in Ruby. Het volgt in grote lijnen het model-view-controller-model. Het streeft naar eenvoud en staat het toe praktische applicaties te ontwikkelen met minder code en minder configuratie dan andere raamwerken. De Ruby-programmeertaal geeft de mogelijkheid tot meta-programmeren waarvan Rails veel gebruikmaakt. Dit resulteert in programmeercode die vaak goed leesbaar is en eenvoudig te begrijpen valt.
